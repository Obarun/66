66-tree

This command handles a set of services.

Interface 

	66-tree [ -h help ] [ -v verbosity ] [ -n new ] [ -a allow user,user,... ] [ -d deny user,user,... ] [ -c current ] [ -E enable ] [ -D disable ] [ -R remove ] [ -C clone ] tree"

66-tree will handle a set of any kind of services into tree. tree will be handled for the current user of the process. Each tree are completely independent from each other.

Options

  • -h  : print the help.
  • -v verbosity : increase/decrease the verbosity of the command. 1(Default): Only print error messages. 2: Also print warning messages. 3: Also print debugging messages.
  • -n  : create a new empty tree. tree doesn't exist yet on the system.
  • -a user : allow user to deal with tree. user can be any valid user on the system. user need to be the name of the user. A list of user separated by comma is allowed. This options is used to set the uid,gid of a service database at the compilation time. That's mean that an undefined user is automatically denied for the given tree. By default the current user of the process is automatically allowed when the tree is created.
  • -d user : deny user to deal with tree. user can be any valid user on the system. user need to be the name of the user. A list of user separated by comma is allowed.
  • -c  : make tree as current. The majority of the 66 tools have a -t options used to specify the tree to use. It can be really annoying to specify this options for each command executed. When you set tree as current all other 66 tools know the tree to use by default and avoid to specify it each time.Also, you can see which tree is the current one with the 66-info tool.
  • -E  : enable tree. This options allow the 66-all tool to know which tree need to be started. If tree is enabled, all services into tree will be started when you use the 66-all tool.
  • -D  : disable tree. This is the strict reverse of the -E options.
  • -R  : Remove tree. Delette tree from the system. This options cannot be reversed, you will not able to retrieve any information of the tree after the use of this options.
  • -C clone : make a strict copy of tree named clone. This is usefull to create a backup of tree or to start a new tree from an existing one. clone doesn't exist yet on the system.
