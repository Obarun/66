66

What is it ?

66 is a collection of tools builded around s6 and s6-rc programs to facilitate the implementations and manipulation of services file on your machine. It is meant to be a toolbox for service declaration, implementation and administration, and that can be assembled together to achieve powerful functionality with a very small amount of code.

Examples of things you can do by assembling together several programs provided by 66 :

  • Frontend service files declaration.
  • Handling all kind of services coming from s6 and s6-rc program.
  • Backup a complete set of divers sort of kind of services.
  • Controlling the environment of the init file.
  • Controlling permissions use of service and scandir or a complete set of services.
  • Nested supervision tree.

The 66 documentation tries to be complete and self-contained; however, if you have never heard of s6/s6-rc programs, you might be confused at first. See the skarnet program documentation to understand the basics in particular the s6 and s6-rc programs.

Miscellaneous

Implementations and handling services file coming from s6 and s6-rc programs can be complex and difficult to understand, that's why 66 tools was born.

Why "66" ?

s6 + s6-rc, so 66. Moreover 66- is really quick and easier to write with the keyboard numpad :).

Also, 66 is a nice command name prefix to have: it identifies the origin of the software, and it's short. Expect more use of 66- in future obarun.org software releases. And please avoid using that prefix for your own projects.
